使用 Z-Image-Turbo 模型来外扩图片

发表于 2025-12-29

无用的前言

先前阿里开源了一个新的文生图模型 Z-Image-Turbo，能在常规的消费级显卡上执行8/9步就能生成一张逼真的图片。
一直对这个保持着关注，直到它的 Controlnet 模型版本也更新的差不多了，终于决定上手把玩一下。
本文编写时，已经推出了Z-Image-Turbo-Fun-Controlnet-Tile-2.1-8steps.safetensors版本，并且好像没有搜索到 Z-Image-Turbo 外扩图片相关的教程。
因此自己简单琢磨一下，写出来作为补充

模型下载

首先，既然你找到了这篇文章，我姑且认为你已经不是完全两眼清澈的小白了。你应该至少也通过 Z-Image-Turbo 生成了合适的图片，想要对这张图片进行外扩。
注意，这里说的是外扩，不是图片超分。
图片超分基本都在用SeedVR2，这个网上教程一大堆，我就不再多说。
而图片外扩，本质就是扩大画布之后，将扩大的部分遮罩住，然后在遮罩区域进行局部重绘。
因此图片外扩本质就是局部重绘的功能延申。

Controlnet

那既然本质是局部重绘，那当然是需要 controlnet 模型了。从 Hugging Face 的alibaba-pai/Z-Image-Turbo-Fun-Controlnet-Union-2.1仓库，找到Z-Image-Turbo-Fun-Controlnet-Tile-2.1-8steps.safetensors并下载。
下载的文件，放在目录models/model_patches下，没有这个目录就新建一个。

底模

底模选择量化版的，从 Hugging Face 的T5B/Z-Image-Turbo-FP8仓库，找到z-image-turbo-fp8-e4m3fn.safetensors并下载。
下载后放在目录models/unet下。
你用你自己文生图时的其它底模也可以，只要是 Z-Image-Turbo 系列的就行。

VAE 和 CLIP

直接用你之前的就行。
没有的话，自己去下载个qwen_3_4b.safetensors放到models/text_encoders。
VAE基本通用的，没有的话从 Tongyi-MAI/Z-Image-Turbo 里面找到 vae/diffusion_pytorch_model.safetensors并下载，下载后放在models/vae目录里。
我这里是下载了diffusion_pytorch_model.safetensors并改名成z-image-turbo-vae.safetensors。

需要一个新的 ComfyUI 节点ZImageFunControlnet，这个节点在我写这篇文章的两周之前就已经在官方内置了。没有这个节点的话自己更新一下。

双击画布空白的地方，选择节点加载图像。随后，连接加载图像到外补画板（ImagePadForOutpaint）节点。后续操作都基于这个外补画板来进行。

通过字典接口获取映射关系

在较为复杂的场景中，一般而言可以通过字典接口获取映射关系，以提高灵活性和可维护性。

如上所示的代码。首先，我们通过字典接口获取映射关系数据。在这个示例中，我们模拟了一个字典接口，该接口返回一个包含键值对的数组。然后我们使用useEffect钩子在组件挂载时调用这个接口，并将返回的数据格式化成一个字典对象。

这种方法的主要优点在于，字典映射文案与前端代码解耦。这意味着我们可以在不重新部署前端代码的情况下，通过更新接口返回的数据来修改映射关系。此外，这种方法还支持一些更复杂的业务场景，例如作为一个标签系统，允许用户在业务流程中动态增删改字典数据。

然而，这种方法也存在一些缺点。首先，字典数据不仅仅在表格中使用，如果表格包含搜索下拉框，搜索功能同样需要字典数据。这意味着前端需要进行状态提升，将字典数据提升到更高层级的组件中（或者Redux这样的状态库中）。此外，在复杂场景中，这种处理方式可能会导致代码难以组织和维护。例如，如果系统需要根据不同租户类型加载不同的字典数据，代码中会充斥各种条件判断，增加了代码的复杂性。

我再加点难度：如果表格的字段是用户自定义显示隐藏的，在用户隐藏字段时无需获取字典数据，而在用户显示字段时才需要获取，这意味着字典的获取应该是实时可变的。这进一步增加了实现的复杂性。

综上，通过字典接口获取映射关系的方法在灵活性和可维护性上有明显优势，但在复杂业务场景中，可能会带来代码复杂性和组织上的挑战。

这个文件的内容比较简单。主要是通过useActive获取到按钮的高亮状态，然后就着remirror的CommandButton和antd的图标封装了个首行缩进按钮。

里面的ToggleBoldButton自带的按钮，是用来对比效果的。

使用带有首行缩进的菜单，同时本地化

还记得@command修饰器注入的那个id吗？将它通过i18n.load加载进去就行了。

至此，一个本地化的、样式交互效果和remirror自带的工具栏按钮一样的首行缩进按钮就扩展完成了。
