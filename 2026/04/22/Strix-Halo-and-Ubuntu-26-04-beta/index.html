Strix Halo and Ubuntu 26.04 beta

发表于 2026-04-22

无用的前言

最近，购入了一台 AMD AI Max+ 395 128G 的迷你主机，并且当作主力电脑使用了近一个月，大致真理一下使用的心得（折腾记录）。

操作系统

准备一个 Ventoy 系统的 U盘，分别放入 win11 和 Ubuntu 的镜像。
首先，先给电脑装了 win 11，再安装了 Ubuntu 26.04 beta。
双系统的目的是为了万一有不兼容的紧急场景，可以及时回退到 win 11 来保底。实际上近一个月的使用，我暂时没有碰到这样的紧急情况。
因为我自己还有台 NAS，常规的资料可以通过 NFS 挂载到迷你主机上。所以这台迷你主机我使用的是 2T 的固态硬盘，分区是：

我必须说一下，我第一次装 Ubuntu 26 的时候，没有单独划分 家目录/home 的空间，而是将剩下的 1.4T 全部丢 根目录/。
后续我重装 Ubuntu 系统的时候只能被迫清空我放在 家目录/home 下的AI模型😟。
所以，这次我学乖了，分出来 200G 到根目录。并且，装了 TimeShift 来做系统的还原（排除 /home 目录）。

远程控制

必不可少的远程控制。
遗憾的是， Ubuntu 26.04 的桌面环境不再使用 x11，所以国内的什么向日葵之类的远程控制都不能使用。
我之前很喜欢用的远程控制RayLink。考虑到我还在RayLink充了288的会员，我捏着鼻子在 Ubuntu 上安装 x11 的桌面环境：

因此，当我需要远程控制时，我就切换并使用 MATE 桌面环境。

可惜，日子不会这么美好的继续下去：我发现我的系统不会熄屏了。不管怎么排查问题，都没有找到原因。（这里我回顾问题，很有可能是多个桌面环境有不同的熄屏/锁屏逻辑，功能配置互相干扰）
这迫使我重装了 Ubuntu 26 系统。

重装系统了之后，我就主打 Tailscale 内网穿透，并配置微林 10块钱一个月的 DERP 服务器来加速远程连接、提高稳定性。
然后，主要使用系统自带的远程连接，备用 NoMachine 进行兜底。

最后这部分再多说一嘴，Ubuntu 26 里先安装 扩展管理器，再在里面搜索 Allow Locked Remote Desktop 插件来确保锁屏的情况下依旧可以远程登录。我看了这个插件的Github，里面的 metadata.json 文件里说是支持 Ubuntu 26 的 GNOME 50。我也没有自己关了这个插件实践试一下能不能锁屏远程登录，总之有总比没有好:)。

GTT 动态显存

因为我需要考虑到日常使用，我就不玩的那么极端。
我一开始，也是按照这篇教程，在 BIOS 里将显存分配成为 512MB（即保留 512MB 的最小显存）。
考虑到我还需要运行桌面环境，可能还需要打开若干个浏览器页面。我最终决定预览 20G 内存给系统，其它的允许动态分配给显存来允许大模型。
所以自己计算一下，我可以动态分配的显存大小是：128G - 20G = 108G。
考虑到内存页面大小是4k，所以：(108*1024*1024)/4=28311552。
按照教程那样，只不过数值改成我自己需要的数值：

只不过，实际的使用过程中，我发现浏览器在观看视频时，会出现整个系统卡死的情况。
所以我在那次重装 Ubuntu 26 系统的时候，我在 BIOS 里将显存分配成为 1GB。
我自己测试了一下，桌面环境开了几个网页+观看 1080p 的60帧直播，GPU 的显存占用在 800MB 左右。并且后续也的确没有出现整个系统卡死的情况。
我怀疑是某些视频只能软件解码额外消耗了显存，然后浏览器没有正确的使用 GTT 动态显存的机制导致系统卡死。但是还是那句话，我也没有自己重新调回去实践排查问题。所以我在这里多记录了一下，没准真的是这个原因呢？多划分 512M 显存也不碍事。

AI 模型

最后，多说一嘴我使用的模型。
我原本是使用Qwen3-Coder-Next-UD-Q4_K_XL.gguf来完成编码工作。
体验过强化思考能力版本的RYS-Qwen3.5-27B-FP8-XL-Q8_0.gguf，与模仿 Claude-4.6-Opus 思维链的Qwopus3.5-27B-v3-Q8_0.gguf，但是说实话太慢了，可以说一个小修改都要10分钟起步。
所以 Qwen3.6-35B-A3B 发布之后，我就立刻切换到 Qwen3.6-35B-A3B-UD-Q8_K_XL.gguf。
我是以路由模式+配置文件的方式运行的 llama-server，可以参考我 Qwen3.6-35B-A3B 的配置文件：

再通过 ZeroNews 暴露端口给我公司的电脑使用，我现在付费模型（token）+本地模型的组合来完成工作，总算是感觉比较舒适了。
你说我为什么不买 code plan？和我的 http 429 说去吧。

我自己也在 comfyUI 下测试了一下生成图片的性能。

我取消掉这个启动参数，每次生成图片后保留模型在显存里，以便测试出扣去加载时间的真实显卡生图性能。
